❌ Критические проблемы UI/UX
1. Панель инструментов над полотном — главная боль
Проблема
Почему это плохо
Закрывает содержимое
Инструменты перекрывают рабочую область — нельзя видеть, что происходит под ними
Постоянно мешает
При работе с большими зумами панель может закрывать центр композиции
Нет привязки
Плавает непонятно где, неясно как свернуть/переместить
Дублирование
Move/Scale/Rotate/Skew есть и в панели над полотном, и в правой панели TRANSFORM — зачем?
Конкретно на скриншоте: - Панель [Cursor] [Move] [Scale] [Rotate] [Skew] [Free] висит в левом-верхнем углу полотна - Под ней ещё одна панель [Rect] [Ellipse] [Lasso] [Polygon] — тоже над полотном - Это две плавающие панели над рабочей областью!

2. Дублирование функционала (конфликт источников правды)
┌────────────────────────────────────────┐│  ПАНЕЛЬ НАД ПОЛОТНОМ    │  ПРАВАЯ ПАНЕЛЬ   │├────────────────────────────────────────┤│  [Move]                   │  Offset X/Y      ││  [Scale]                  │  Scale X/Y       ││  [Rotate]                 │  Rotation        ││  [Skew]                   │  Skew X/Y        ││  [Free]                   │  (нет аналога)   │└────────────────────────────────────────┘
Проблема: Пользователь не понимает, где «правильно» менять параметры. Если крутанул Rotation в правой панели — кнопка [Rotate] над полотном подсвечивается? Нет? А если нажал [Rotate] — значение в правой панели обновляется?
Результат: Неуверенность, лишние клики, баги синхронизации.

3. Панель «Transform (moveable)» — мёртвый груз
Внизу полотна висит плавающая панелька:
Transform (moveable)x: 0.0 · y: 0.0scale: 1.00 × 1.00rot: 0.0° · skew: 0.0° · 0.0°[Reset] [Exit] [Clear Mask]
Проблемы: - «moveable» — зачем это в UI? Пользователю всё равно, moveable она или нет - Дублирует правую панель — те же значения уже есть в TRANSFORM справа - «Exit» — выход из чего? Из трансформации? Из режима? Неясно - «Clear Mask» — зачем в панели трансформации кнопка очистки маски? Это разные домены

4. Правая панель перегружена и плохо сгруппирована
Текущий порядок:
SCREENTONE 1 — SIMPLEPatternShapeSizeSpacing XSpacing YDensityColorBackgroundAngleTRANSFORMOffset XOffset YScale XScale YRotationSkew XSkew Y[Reset] [Reset Persp.]
Проблемы: - 11 полей подряд без визуальной иерархии — глаз «проваливается» - Color и Background — зачем Background в панели скринтона? Это свойство слоя/полотна, не скринтона - Spacing X и Spacing Y отдельно — обычно это одно значение (квадратная сетка), редко нужно раздельно. Лишнее поле = лишний когнитивный груз - Reset и Reset Persp. — «Persp.» это сокращение? Пользователь может не понять. И зачем две кнопки сброса?

5. Левая панель: Layers и Presets в одной колонке
LAYERS[+ Tone] [+ Image] [+ Solid]Screentone 1  Normal [100%]  [🔒] [▼] [🗑] [✨] [✕]Background  [eye]PRESETSSearch...[All ▼][сетка иконок пресетов]
Проблемы: - Layers занимают ~30% высоты, Presets — остальное. При 10+ слоях Presets вообще уйдут за экран - Пресеты в виде сетки иконок — без названий видны только иконки. «Deleter 10% Dot» и «Deleter 30% Dot» визуально почти идентичны - Нет разделителя между Layers и Presets — визуально сливаются - Кнопки слоя (🔒 ▼ 🗑 ✨ ✕) — мелкие, без tooltip’ов, неясно что делают

6. Top Bar — всё в одну линию без группировки
[GenTonik v2] [New] [Open.ora] [Save.ora] [Import PNG] [Export PNG] [Undo] [Redo] [2000×2000] [DPI 300] [-] [41%] [+] [Fit] [Simple]
Проблемы: - 14 элементов в ряд — нет визуальной группировки (файл / правка / вид / настройки документа) - «Open.ora» и «Save.ora» — расширение в названии кнопки? Зачем? Пользователю всё равно, это внутренний формат - «Simple» — что это? Режим интерфейса? Неясно без контекста - Zoom (- 41% + Fit) — мелкий, без выпадающего списка процентов

7. Bottom Bar — бесполезная строка
[+ Save] [↓] [↑]
«+ Save» — Save уже есть в топ-баре. Дублирование
Стрелки вниз/вверх — для чего? Слои? Пресеты? Неясно без контекста

📊 Сравнение: GenTonik vs Krita vs Figma


1. Что перенять у Krita
1.1. Docker-система (гибкие dockable-панели)
Krita использует концепцию Dockers — панелей, которые можно: - Перетаскивать между левой/правой/верхней/нижней зоной - Свернуть в табы - Вынести во floating-окно - Сохранить раскладку как Workspace
Как это применить к GenTonik:
┌─────────────────────────────────────────────┐│  Menu Bar                                    │├────────┬──────────────────────────┬───────────┤│        │                          │           ││  [Toolbox]│     CANVAS             │ [Tool     ││  [Layers] │     (чистое полотно)   │  Options] ││  [Presets]│                        │ [Color]   ││         │                          │           ││         │  Handles на объекте      │           ││         │  (не панели над полотном)│           ││         │                          │           │├────────┴──────────────────────────┴───────────┤│  Status Bar                                  │└─────────────────────────────────────────────┘
Практическое решение: - Убрать все плавающие панели над полотном - Инструменты выделения (Lasso, Rect, Ellipse, Polygon) — в левый Toolbox (вертикальная иконка-панель) - Параметры текущего инструмента — в правый Tool Options docker (как в Krita) - Layers и Presets — в левый dock, но в разных табах, а не в одной куче
1.2. Tool Options Docker — контекстные параметры инструмента
В Krita при выборе инструмента его параметры появляются в отдельном docker’е:

User interface for Transform tool options discussion - Develop - Krita Artists
Что видим: - Move Tool → показывает X, Y, Position/Rotate/Scale/Shear переключатели - Transform Tool → показывает Filter, Pivot Point, Camera height - Кнопки Reset / Apply внизу
Как перенести в GenTonik:
Текущий GenTonik
Как в Krita
Move/Scale/Rotate/Skew/Free — кнопки над полотном
Один Transform Tool в Toolbox → в Tool Options выбираешь режим (Move/Scale/Rotate/Shear/Free/Perspective)
Все параметры трансформации всегда видны
Показывать только релевантные выбранному режиму
Reset и Reset Persp. — две кнопки
Reset + Apply (как в Krita)
1.3. Handles на объекте, не в UI
В Krita при выборе Transform Tool на объекте появляются рамка с точками — ты тянешь за них прямо на полотне:

Tool options docker completely blank - Basics & Interface - Krita Artists
GenTonik сейчас: панель [Move] [Scale] [Rotate]... над полотном + плавающая панель Transform (moveable).
GenTonik должно быть: - Выбрал инструмент в Toolbox → на объекте появляются handles - Тянешь за угол → свободная деформация - Тянешь за середину стороны → масштаб по одной оси - Тянешь за центр → перетаскивание - Никаких панелей над полотном
1.4. Workspaces (сохранение раскладки)
Krita позволяет сохранять и переключать раскладки интерфейса. Для GenTonik это полезно: - Режим «Редактирование скринтона» — показывать Tool Options + Layers - Режим «Выбор и маскирование» — показывать Selection Tools + Tool Options - Режим «Трансформация» — показывать Transform + Tool Options + Layers
1.5. Pop-up Palette (быстрый доступ к пресетам)
Уникальная фича Krita — правый клик на полотне → круговое меню с кистями/цветами.
Для GenTonik: Правый клик на полотне → быстрое меню: - Пресеты скринтонов - Недавние цвета - Быстрые действия (Duplicate, Rasterize, Delete)
